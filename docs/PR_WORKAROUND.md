# Работа с PR, если ветка уже менялась вне Codex

Платформа Codex не умеет обновлять открытый PR, если ветка успела измениться вне Codex. Чтобы без ошибок выкатывать изменения в GitHub, сделайте новый PR с актуальной ветки.

## Быстрый план
1. Убедитесь, что локальная ветка содержит нужный коммит.
2. Создайте новую ветку от текущего состояния:
   ```bash
   git checkout -b fix/codex-pr-refresh
   ```
3. Запушьте её в свой репозиторий:
   ```bash
   git push origin fix/codex-pr-refresh
   ```
4. Откройте новый Pull Request из этой ветки. Старый PR, созданный до ручных правок, можно закрыть.

## Почему так
Сообщение вида «Codex в настоящее время не поддерживает обновление PR, обновляемых за пределами Codex. Создайте новый PR» означает, что текущий PR уже менялся вручную. Единственный надёжный способ продолжить — открыть новый PR с актуальными изменениями.
